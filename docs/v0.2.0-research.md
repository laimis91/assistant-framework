## Gap Analysis: Current Framework vs. Landscape

### Strengths (keep)
1. Structured workflow with right-sized ceremony
2. Composable skills that work standalone
3. Multi-agent dispatch with role-based access control
4. Hook system for automated behaviors
5. Memory graph MCP server
6. Cross-platform support (Claude, Codex, Gemini)
7. Invisible workflow principle

### Gaps

#### 1. Memory is Too Primitive
- No semantic search (only text matching)
- No memory decay (stale memories persist forever)
- No automatic capture (relies on agent discipline)
- No compression of chat history into memory
- No cross-machine sync

#### 2. No Reflexion / Self-Improvement Loop
- No post-task reflection
- No cumulative learning about what approaches work
- No strategy evolution

#### 3. No Proactive/Event-Driven Behavior
- Entirely reactive
- No repo watching, CI monitoring, proactive suggestions

#### 4. No Documentation Generation Skill
- Biggest stated user weakness with no dedicated skill

#### 5. No General Visual/Diagram Generation
- Unity art skills exist but nothing for architecture diagrams, flowcharts

#### 6. No Structured Ideation Skill
- creative.md thinking tool is generic, no codebase-aware brainstorming

#### 7. No Project Management Integration Skill
- ClickUp MCP exists but no orchestrating skill

#### 8. No Onboarding Skill for New Projects
- No systematic "learn this codebase" capability

## Unique Position

No other framework combines workflow + review + memory + thinking tools + security in one composable system. The next step is making it self-improving.
