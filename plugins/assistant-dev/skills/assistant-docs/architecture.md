# Architecture Documentation Generator

## Protocol

### Step 1: Map the System

Use Code Mapper or manual exploration to identify:

- **Entry points**: Program.cs, Startup, Main, endpoints
- **Layers**: presentation, application, domain, infrastructure
- **Key abstractions**: interfaces, base classes, shared contracts
- **Data flow**: request → handler → service → repository → database
- **External dependencies**: databases, APIs, message queues, file systems
- **Configuration**: appsettings, environment variables, feature flags

### Step 2: Identify Architecture Pattern

Determine what pattern the project uses:
- Clean Architecture / Onion
- N-tier / Layered
- Vertical Slices
- CQRS
- Microservices
- Monolith
- Hybrid

State the pattern and note any deviations.

### Step 3: Generate

Produce a document with these sections:

```markdown
# Architecture Overview

<!-- generated by assistant-docs: architecture -->

## System Context

[1-2 sentences: what the system does and who uses it]

## Architecture Pattern

[Pattern name] with [notable deviations]

## Component Diagram

[Mermaid diagram showing major components and their relationships]

## Layer Responsibilities

### [Layer Name]
- **Purpose:** [what this layer does]
- **Key types:** [important classes/interfaces]
- **Depends on:** [what it references]
- **Depended on by:** [what references it]

## Data Flow

[Mermaid sequence diagram for the most common operation]

## Key Design Decisions

| Decision | Rationale | Trade-offs |
|---|---|---|
| [decision] | [why] | [what was given up] |

## External Dependencies

| System | Purpose | How Connected |
|---|---|---|
| [database] | [purpose] | [EF Core / Dapper / raw SQL] |

## Configuration

| Setting | Purpose | Where Set |
|---|---|---|
| [setting] | [what it controls] | [appsettings / env var / secret] |
```

### Step 4: Include Diagrams

Generate Mermaid diagrams for:
1. **Component diagram** — boxes and arrows showing major components
2. **Data flow** — sequence diagram for primary operation
3. **Dependency graph** — which projects/assemblies reference which

If `assistant-diagrams` skill is available, delegate diagram generation to it.

### Step 5: Verify

- Every component mentioned exists in the codebase
- Every dependency listed is real (check .csproj references, using statements)
- Data flow matches actual code path (trace it)
- No hallucinated components or relationships

### Output Location

- `docs/architecture.md` (or update existing)
- Mark generated: `<!-- generated by assistant-docs: architecture -->`
