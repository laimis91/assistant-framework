# Playtest Feedback Template

Use this structure to capture feedback from a playtest session. The user doesn't need to fill every field — capture what they mention and ask follow-up questions for critical gaps.

## Quick Feedback (Minimum)

Ask the user three questions:
1. **What felt good?** (What was fun, satisfying, or worked well)
2. **What felt bad?** (What was frustrating, confusing, or broken)
3. **What's missing?** (What did you expect that wasn't there)

## Detailed Feedback Template

### Feel & Controls
- Movement: [too fast / too slow / just right / floaty / snappy]
- Responsiveness: [inputs feel immediate / delayed / inconsistent]
- Camera: [comfortable / too close / too far / disorienting]

### Combat & Weapons
- Damage feedback: [satisfying / unclear if hitting / too subtle]
- Weapon variety: [enough choices / too few / too many similar options]
- Enemy threats: [readable / confusing / unfair / too easy]

### Progression
- Upgrade choices: [meaningful / unclear / too many bad options]
- Power curve: [feels like getting stronger / feels flat / too slow / too fast]
- Run length: [too short / too long / about right]

### Visual Clarity
- Can tell player from enemies: [yes / sometimes / no]
- Can see projectiles: [yes / hard to see / gets lost in chaos]
- Can read UI: [yes / too small / blocked by gameplay]

### Fun Factor (1-5)
- Core gameplay loop: [1-5]
- "One more run" feeling: [1-5]
- Best moment: [describe]
- Worst moment: [describe]

### Bugs Encountered
- [List any bugs with steps to reproduce]

### Suggestions
- [Freeform suggestions from the player]
