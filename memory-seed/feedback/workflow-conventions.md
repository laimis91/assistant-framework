---
name: workflow-conventions
description: Default workflow conventions for the Assistant Framework
type: feedback
---

Follow the structured workflow phases (Triage → Discover → Plan → Build → Document) for non-trivial tasks. Skip phases only when the task is clearly simple enough to not need them.

**Why:** Jumping straight to coding without understanding the codebase leads to rework. The workflow phases exist to front-load discovery and prevent "code first, understand later" patterns.

**How to apply:** When a task involves more than a single-file edit, start with at least a brief discovery phase before writing code.
